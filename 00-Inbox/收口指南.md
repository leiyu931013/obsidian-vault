# 📥 Inbox 收口指南

> **目的**: 避免 Inbox 变垃圾山
> **节奏**: **每天 12:00 / 22:00 各处理一次**（3 分钟）
> **零容忍**: 超过 10 条就开始焦虑，超过 20 条必须当天清

## 🚀 快速收口 (3 个入口)

### 1. 终端
```bash
inbox "研究一下 Hermes 跟 OpenClaw 哪个更适合"
# 软链: /Users/sunny/bin/inbox
# 真实: /Users/sunny/.hermes/scripts/inbox_capture.sh
```

### 2. 微信
跟 Hermes 说：
> "记录: 群里有人推荐 Firecrawl，可以抓任意网页转 LLM 友好格式"

Hermes 会调 `inbox_capture.sh` 追加到 `00-Inbox/Inbox.md`，带 `(wechat)` 来源标签。

### 3. Obsidian GUI
- `Cmd + P` → 敲 `Open Inbox`
- 或左侧文件树点 `00-Inbox/Inbox.md`

## 📋 收口流程 (3 分钟 / 次)

### Step 1: 打开 Inbox
- 终端: `inbox` (无参数, 自动开 Obsidian)
- GUI: 直接点 `00-Inbox/Inbox.md`

### Step 2: 扫一遍条目
对每条问自己：

| 问题 | 处理 |
|---|---|
| 是个**完整想法**值得记 | → 移到 Areas/Resources/Projects |
| 是**待办** | → 移到 `01-Daily/今天.md` 的 TODO 段 |
| 是**参考资料/链接** | → 移到 `04-Resources/` |
| 是**已实现/过期** | → 删 |

### Step 3: 搬移（Obsidian GUI）
- 拖到目标目录
- 或: 选中段落 → Cut → 切到目标文件 → Paste
- 加 `[[wikilink]]` 反向引用

### Step 4: 删 Inbox 里的原条目
- 留下: 时间戳+标签
- 删掉: 实际内容（已搬到别处）

## 🏷️ 来源标签

Inbox 里的每条都有来源标签：
- `#terminal` — 你在终端输的
- `#wechat` — 你在微信里跟 Hermes 说的
- `#spark` — spark 上自动收口
- `#cron` — 定时任务提醒
- `#公众号` — 公众号后台产生

## 🤖 自动收口

| 频率 | 源 | 写到哪 |
|---|---|---|
| 每 6h | spark 设备状态 | `99-Spark-Sync/设备状态.md` |
| 每 6h | 公众号发文统计 | `04-Resources/公众号发文明细.md` |
| 每 6h | vault 自身统计 | `📈 vault 统计.md` |
| 每 6h | Inbox 满 5/10/15... | Inbox 自动加一条提醒 |

## ⚠️ 黄金法则

1. **进 Inbox < 30 秒**: 别在捕获时编辑，先进 Inbox 后面再处理
2. **每天处理 2 次**: 12:00 + 22:00，3 分钟一次
3. **每条 < 5 秒决断**: 要么移到别处，要么删
4. **不处理就清空**: 留着的都是负担
5. **2 周没动的 = 没价值**: 大胆删
