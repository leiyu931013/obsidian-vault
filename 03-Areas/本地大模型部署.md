# 🎨 本地大模型部署

> ComfyUI + Ollama 是主要工具。GB10 130GB VRAM 是大杀器。

## 📚 现在知道的

### ComfyUI (图像)
- **版本**: 0.25.1 (spark 端)
- **路径**: `/home/new/git/ComfyUI-0.25.1/`
- **模型**:
  - UNET: `models/diffusion_models/z_image_turbo_bf16.safetensors` ⚠️ 在 diffusion_models/ 不是 unet/
  - CLIP: `models/text_encoders/qwen_3_4b.safetensors`
  - VAE: `models/vae/ae.safetensors`
- **节点**: UNETLoader, CLIPLoader, VAELoader, ModelSamplingAuraFlow, KSampler (res_multistep)
- **工作流**: prompt → KSampler (8 steps, cfg=1.0) → VAEDecode → SaveImage
- **首推模型**: Z-Image-Turbo (步骤 8 即可)
- **系统 systemd 守护**: `~/.config/systemd/user/comfyui.service`
- **Linger**: yes (用户未登录也跑)

### Ollama (LLM)
- spark 上没装（待部署）
- GB10 130GB VRAM 跑 qwen3 235B / deepseek-v3 都能跑
- 本地 0 网络依赖，离线可用

### 已知坑
- ComfyUI 0.25.1 的 `DiffusionModelLoader` 节点不存在（用 UNETLoader）
- alembic 数据库 init 报"Failed to initialize database"——是文档截断，**无害**
- 批量请求时 iLink 限流 (30s cooldown)

## 📈 关注

- spark 上加 fallback LLM（目前只有 minimax-cn 单一 provider）
- ComfyUI 升级路径
